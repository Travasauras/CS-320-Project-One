How can I ensure that my code, program, or software is functional and secure?
Using Black-box and White-box testing techniques and adhering to industry standards to ensure that my code is functional, 
meets the requirements, and easily read and modified. 

How do I interpret user needs and incorporate them into a program?
I review the requirements of the system and the possible interactions between data given by the end user and the code I have 
written.  I then test every possible problem that may arise and implement my code into program. 

How do I approach designing software?
I approach it in somewhat of a pessimistic way.  Not in the sense that I don't enjoy it, but rather applying Murphy's law to the
program itself.  If anything can go wrong, will go wrong.  With this in mind, I look for the can go wrongs to ensure that they
can't go wrong.
